# 📋 DDD — Data Design Document

Template para times de **Analytics Engineering**, com foco em projetos de dados.

🔗 **[Acesse o template →](https://jptudobem42.github.io/data-design-document/)**

---

## O que é

Um **Data Design Document (DDD)** é um documento estruturado que centraliza tudo o que precisa estar documentado antes de construir um modelo de dados: contexto de negócio, fontes, modelagem, testes e impacto em modelos existentes.

Este template cobre as 8 seções essenciais do processo:

| # | Seção | Obrigatoriedade |
|---|---|---|
| 1 | Contexto e motivação | Obrigatório |
| 2 | Definição e escopo | Obrigatório |
| 3 | Fontes de dados e dependências | Obrigatório |
| 4 | Modelagem e design técnico | Obrigatório |
| 5 | Testes e qualidade de dados | Obrigatório |
| 6 | Impacto em modelos existentes | Obrigatório |
| 7 | Documentação e exposição | Recomendado |
| 8 | Alternativas consideradas | Opcional |

## Contexto dos exemplos

Os exemplos fictícios incluídos no template foram escritos para o domínio de **saúde hospitalar**, especificamente o fluxo de **internações originadas no Pronto Atendimento**, incluindo:

- Triagem pelo Protocolo de Manchester
- Taxa de internação por classificação de risco
- Tempo médio de espera por turno
- Saídas a pedido (AMA)
